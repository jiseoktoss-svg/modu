# UX 라이팅 가이드

modu의 모든 사용자 노출 문구(화면 본문, 버튼, 라벨, 토스트, 에러 메시지, 공유 텍스트, 데모 데이터)에 적용한다.
새 페이지나 기능의 문구를 만들 때 이 문서를 기준으로 쓰고, 기존 문구를 고칠 때도 이 규칙에 맞춘다.
(토스 UX 라이팅 가이드라인을 기반으로 하며, 마지막 절에 modu 프로젝트 고유 결정 사항을 정리했다.)

## 1. 해요체

상황·맥락을 불문하고 모든 문구는 해요체로 쓴다. "~습니다/합니다"는 쓰지 않는다.

- ❌ `회의명은 최대 20글자까지 입력할 수 있습니다.`
- ✅ `회의명은 최대 20글자까지 입력할 수 있어요.`

## 2. 능동적 말하기

수동형 대신 능동형 문장을 쓴다(예외는 아래 참고).

- **됐어요 → 했어요**: ❌ `회의 생성에 실패했습니다.` → ✅ `회의를 만들지 못했어요.`
- **'~었' 빼기**: ❌ `회의 시간이 확정되었어요` → ✅ `회의 시간이 정해졌어요`
- **동사 바꿔쓰기**: 더 자연스러운 동사가 있으면 교체한다. ❌ `변동될 수 있어요` → ✅ `바뀔 수 있어요`

## 3. 긍정적 말하기

"안 돼요/없어요"보다 "~하면 할 수 있어요"로 뒤집어 쓴다.

- ❌ `필수 참석자가 없어 회의 시간을 확정할 수 없습니다.`
- ✅ `필수 참석자가 있어야 회의 시간을 정할 수 있어요.`

에러 메시지는 해결 행동을 먼저 알려준다.

- ❌ `응답 마감 날짜가 올바르지 않습니다.` → ✅ `응답 마감 날짜를 다시 확인해 주세요.`
- ❌ `회의 길이가 근무 시간보다 깁니다.` → ✅ `회의 길이를 근무 시간 안으로 줄여 주세요.`

권한/조건 안내는 부정 나열 대신 되는 조건만 말한다.

- ❌ `이 응답을 수정할 권한이 없습니다. 처음 제출한 브라우저에서만 수정할 수 있어요.`
- ✅ `처음 제출한 브라우저에서만 수정할 수 있어요.`

## 4. 캐주얼한 경어

과도한 경어('~시겠어요?', '~시나요?', '께', '계시다', '여쭈다')를 쓰지 않는다.

- **'~시' 빼기**: ❌ `그래도 선택하시겠습니까?` → ✅ `그래도 선택할까요?`
- **'께' → '에게'**: ❌ `함께할 분들께 보내기만 하면` → ✅ `함께할 분들에게 보내기만 하면`
- '~시'를 빼서 어색하면 파악하려는 정보를 주어로 문장을 다시 쓴다. 예: `어떤 이유로 신고하시나요?` → `신고 이유를 선택해주세요`

'드려요'(찾아드려요, 보여드려요)는 과도한 경어가 아니므로 써도 된다.

## 5. {명사}+{명사} 쓰지 않기

한자어 명사는 풀어서 동사로 쓴다.

- ❌ `참석자 저장에 실패했습니다.` → ✅ `참석자를 저장하지 못했어요.`
- ❌ `적절한 시간값을 입력해주세요.` → ✅ `시간을 올바르게 입력해 주세요.`
- ❌ `참석자 전달 링크` → ✅ `참석자에게 보낼 링크`
- ❌ `변동 가능성이 있습니다` → ✅ `순위가 바뀔 수 있어요`

풀어쓰기 어려우면 '{명사}가 {명사}해서' 형태로라도 푼다.
예: `잔액부족으로 구매하지 못했어요` → `잔액이 부족해서 구매하지 못했어요`

## 예외 규칙

**수동형을 써도 되는 경우**

- 서비스 종료·기간 만료처럼 주어(종료, 만료)를 강조할 때. 단, 주기적으로 종료가 반복되는 곳에는 '종료돼요'를 쓰지 않는다.
- 사용자 행동에 따라오는 결과(인과)를 알려줄 때. 예: `투표하면 회의는 저절로 정해져요`
- 정보 수집 안내 등 민감한 상황에서 사용자를 안심시킬 때.
- '되어요'는 항상 '돼요'로 줄여 쓴다.

**부정형을 써도 되는 경우** — 쓸 수 없는 이유를 함께 안내한다.

- 정책상 서비스 자체를 쓸 수 없을 때. 예: `이미 확정된 회의라 응답할 수 없어요.`
- 일부 기능만 쓸 수 없을 때(어떤 기능인지 명확히). 예: `필수 참석자만 후보 시간대에 투표할 수 있어요.`
- 빈 상태의 명확한 안내. 예: `검색 결과가 없어요.`, `표시할 추천안이 없어요.`
- 안심시키는 부정문. 예: `실제 이메일이나 캘린더 초대는 보내지 않아요.`

## modu 고유 결정 사항

- **동적 값 뒤의 어미**: 문장 빌더처럼 사용자 입력값 뒤에 서술격조사가 붙으면 `lib/korean.ts`의 `hasBatchim()`으로 '이에요/예요'를 분기한다. 값이 고정 단어로 끝나면(예: "N명", "N시간/분") 받침에 맞는 어미를 고정해도 된다. '이며'는 쓰지 않고 '이고'를 쓴다.
- **"회의가 만들어졌어요"**(완료 화면 타이틀): 결과 강조를 위한 수동형으로 예외 유지.
- **"응답 마감"**: 서비스 용어로 유지한다(라벨·에러 공통). 문장에서 자연스럽게 풀 수 있으면 푼다.
- **상태 라벨**(필수/선택, 가능/불가능/미응답 등): 뱃지용 축약 라벨은 명사형 그대로 둔다.
- **데모 안내**: 실사용 화면이 아님을 알릴 때는 "~데모 영역이에요", "실제 사용 화면에는 보이지 않아요" 톤을 쓴다.
- 개발자용 문구(콘솔 경고, 주석)는 이 가이드의 대상이 아니다.
- 서버 검증 문구를 바꿀 때는 테스트(`test/*.test.ts`)가 문구를 단언하는지 확인하고 같이 갱신한다.
